About

The Recipe Search website is designed to provide users with a convenient way to explore and discover various recipes. As a guest user, you can freely search and browse through a wide range of recipes available in our system. Once you create an account and log in, you gain access to additional features, including the ability to contribute your own recipes to our growing collection. Join our community of food enthusiasts and start sharing your favorite recipes with others. Happy cooking!

UI Highlights

  • Layout

    The template is designed with a responsive layout optimized for three different devices:
    1. Desktop: The layout expands up to a maximum width of 90em (equivalent to 1440px, assuming a default font size of 16px). The main content is contained within a 990px frame, providing a clean and readable white background.
    2. Tablet: The layout adjusts to a maximum width of 100% when the screen size of the device is less than or equal to 48em, ensuring optimal display and usability on tablets.
    3. Mobile Phones:The layout adapts to a maximum width of 100% for screen sizes smaller than or equal to 30em, ensuring a seamless and user-friendly experience on mobile phones.
    These responsive design choices are implemented using media queries to ensure the template's adaptability and readability across different devices.

  • Category Carousel on the Home Page

    The category carousel on the home page is implemented using a sample slider obtained from w3schools. The original slider was customized to match the site's layout by adjusting the styles and timing of the JavaScript code.

  • Persistent Footer Placement

    To ensure that the footer remains at the bottom of the page regardless of the content height, several steps were taken:
    • The main div element, which serves as the parent of the footer, was set to a height of 100vh (viewport height). This ensures that the main content area occupies the available vertical space on the page.
    • The footer was styled with `position: absolute` and `bottom: 0`, allowing it to be positioned at the bottom of the viewport.
    • To prevent the footer from overlapping the content on other pages, additional modifications were made. Footer-placeholder elements were added to reserve space equal to the height of the footer, ensuring that the content remains visible and unobstructed.

  • Add ingredients and method steps in recipe create form

    Created an algorithm in js to append form fields when user click on add and delete ingredients and method steps of recipe create form

Exploring the Back-end Implementation

  1. Architecture

  2. Python libraries

    • Flask: I selected Flask as the framework for its versatility, simplicity, and extensive community support. Flask utilizes the Jinja2 template engine, enabling the creation of dynamic HTML templates. With Flask and Jinja2, I developed a robust and scalable application that meets the project requirements effectively.
    • Sqlite: I decided to use Sqlite as the database for this project because it is a database system that I learned and became familiar with during the semester 1. Since I already had knowledge of how to work with Sqlite, it made sense for me to leverage that expertise and apply it to this project. By choosing Sqlite, I was able to apply the skills and understanding I gained from studying the database module at university.
    • SQLAlchemy: I chose to utilize SQLAlchemy as the ORM (Object-Relational Mapping) library for this project. SQLAlchemy provides a powerful and flexible way to interact with databases using Python. It simplifies the process of building and executing database queries, allowing for easier and more efficient development. By leveraging SQLAlchemy, I was able to take advantage of its rich set of features and functionality, making the development process smoother and more straightforward.
    • Flask-Migrate: I used Flask-Migrate to make managing database changes easier. It allowed me to update the database structure without writing complex SQL queries directly. Instead, I could define the changes using simple Python code. Flask-Migrate then handled generating the necessary SQL statements for me. It also provided a convenient way to run seed scripts, which helped me populate the database with initial data effortlessly. Thanks to Flask-Migrate, I could efficiently manage and update the database for the website without getting tangled up in intricate database queries.
    • Flask-Login: I utilized Flask-Login to simplify the authentication process in my application. It provided an easy-to-use interface for handling user login and session management. With Flask-Login, I could securely handle user authentication, login sessions, and access control with minimal code. It abstracted away the complexities of session management and provided useful features like remembering users across sessions and protecting routes that required authentication. Flask-Login made it straightforward to implement authentication functionality in my application, saving me time and effort.
    • Flask-WTF: I used Flask-WTF to simplify form management in my application. It provided convenient features for handling forms, including validation and protection against CSRF attacks. With Flask-WTF, I could easily define form fields, specify validation rules, and ensure the security of form submissions. It made form handling straightforward and enhanced the overall user experience.
    • Pillow: I used this library to get the size and width of the uploaded image to validate
    • Vertual environment I used this for managing dependencies in this project. While I initially intended to automate the dependency installation process similar to the package.json in Node.js, time constraints prevented further exploration. However, I discovered a solution: using pip install -r requirements.txt, which offers a convenient approach.

  3. Decorators and macros

    • Decorators: I created a decorator guest_users_only to hide some rout endpoints from login users
    • Macros: I have developed a set of macros that efficiently centralize the rendering of UI elements, enhancing the ease of management and improving overall workflow.
